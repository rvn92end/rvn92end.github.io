Installing Conduit

Before installing Conduit, make sure you understand the basics of and become familiar with Google Cloud Platform. For information needed to get started with GCP, refer to Getting Started with GCP.

Based on your scenario, installing Conduit may require one-time preparation. Before you get started, refer to the prerequisites below.

 

Permissions

Default permissions are turned on in Vertex AI. All permissions should be defaulted correctly. (Check before shipping to make sure this is true. If not, amend this section to explain what is required.)

 

Installation steps

In a command window on your notebook, enter the following commands:

 

Step

Command

Result

1.

git clone git@github.aetna.com:dso/conduit.git

 

The Conduit GitHub repo is cloned to your notebook.

2.

python conduit/setup.py

 

The Conduit terminal user interface (TUI) program is run. Directory structure is created.

3.

conduit

 

The Conduit TUI is launched in the command window.

Once the Conduit program launches, follow the instructions below to use Conduit to build a pipeline.

Creating a pipeline

This section explains how to use the Conduit terminal user interface to assemble your pipeline, using templated files and inputs you provide. Once the pipeline is assembled, it is added to the GitHub repository referred to in the Prerequisites section above.

 

Quickstart

Follow these instructions to use the Conduit interface to create a pipeline:

  1. Enter conduit in the command window to launch Conduit.


  2. Outside of the Conduit interface, navigate to the files outlined below, open, and edit to enter the required information, as dictated by the comments in the files:

    • src\feature_names.json

    • src\batch_prediction_data.sql

    • src\train_data.sql

    • src\ground_truth_data.sql

    • custom_container\training\train.py

  1. Save the files in the existing location, with the existing names.

  2. In the Conduit interface in the command window, click Create Pipeline to begin.

    Note: As you move through the Conduit interface, inputs are saved at regular intervals. If you must abandon the process (use Q to quit), you can relaunch Conduit and click Resume on the first screen to pick up where you left off.

What happens next

Conduit configure the ML pipeline, trains the models, and executes batch processes using the information provided. You may see this screen before the final screen:



If an error occurs, you are notified. When the build is complete, the pipeline is stored in the Conduit GitHub repository you created for the project.

Click Close to exit Conduit. (You can click back to edit to return to the questions if necessary.)

Create a PR, merge, and deploy

At this point, to deploy your model, do the following:

  1. Create a pull request:

    1. In GitHub, navigate to the main page of the repository you created at the start of this process.

    2. In the "Branch" menu, choose the branch that contains your commits.

    3. Above the list of files, in the yellow banner, click Compare & pull request to create a pull request for the associated branch.

    4. Use the base branch dropdown menu to select the branch you'd like to merge your changes into, then use the compare branch drop-down menu to choose the topic branch you made your changes in.

    5. Type a title and description for your pull request.

  2. When the pull request is approved, merge the your branch.

  3. Deploy the pipeline to Vertex AI.

What Conduit is doing

After you click Build, behind the scenes Conduit performs the following actions:

  • Generates a Vertex Pipeline configuration file that includes:

    • GCP information (e.g., Project, Bucket, Service Account, etc.)

    • Model information (e.g., model name, model description, target metric, etc.)

    • Pipeline parameters (e.g., scheduling instructions, endpoint creation, etc.)

  • Generates a production-ready Vertex Pipeline complete with:

    • Model monitoring (e.g., feature drift/skew detection and label drift detection)

    • Continual learning (i.e., OOS feedback loop that retrains the model when performance degrades)

    • CI/CD integration with necessary Jenkins files

    • Model Registry integration for tracking model versions

    • Artifact Registry integration for tracking containers, pipelines, endpoints, etc.

  • Generates a Cloud Composer pipeline

  • Stages a pull request (PR) that is ready to be committed to the main branch for deployment

Frequently asked questions

How does Conduit help in configuring ML pipelines, training, and batch processes? 

Conduit’s terminal user interface (TUI) guides users through the entire process of configuring their ML pipeline, training their models, and executing batch processes. It simplifies these tasks by taking the minimum number of user inputs required to deploy and manage a pipeline on Vertex AI. 

 

Will pipelines generated by Conduit leverage Vertex AI’s visualization and monitoring capabilities?  

Yes. Conduit generated pipelines can be visualized, observed, and monitored in the Vertex AI console.   

 

How does Conduit improve reproducibility and collaboration?

Conduit stores all necessary information, including Python library requirements, in a Docker file. This approach ensures that all team members can access and use the same environment, making it easy to reproduce results and collaborate on ML projects.

By using Conduit to create Vertex Pipelines, your ML pipeline will be fully reproducible by design. Models are stored and versioned in the Model Registry and all artifacts (e.g., containers, pipelines, etc.) are stored in the Artifact Registry. Along with the code in your repo, this will promote transparency and discoverability. Additionally, training data snapshots are stored in GCS and all your training and prediction outputs are captured in standardized schemas in Big Query which can be readily queried for downstream consumption and/or reporting.

 

How does Conduit handle cost center and resource accounting?

While using Conduit, or more generally GCP, resource usage is tracked at the tenant level so that costs can be appropriately compartmentalized and shared (e.g., transferred out to business partners).

 

Which programming languages does Conduit support for building ML models?

Currently, Conduit supports only Python for building ML models. In the future, R will be supported as well.

 

Will all of Vertex AI’s services be fully enabled in DEV?

No, the intention is to focus on unincumbered development in the lower environment whereas the higher environments (e.g., TEST and PROD) are oriented toward ML pipeline orchestration.

 

Can users looking for a custom experience directly leverage the KFP and/or TFX APIs to create their own pipelines?

No, currently Vertex AI services (apart from Vertex managed notebooks) are not enabled in the DEV environments by design. This is to ensure that maintained consistency for all of production-deployed ML pipelines from compliance, governance, reliability, and observability perspectives. If your ML pipelines pattern(s) fall outside of what is currently supported in Conduit, please reach out to EnterpriseMLPlatform@aetna.com.

Refer to the ML Platform Roadmap.
