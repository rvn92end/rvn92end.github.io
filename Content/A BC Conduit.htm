Welcome to Conduit 

Conduit is a Python package and terminal user interface (TUI) designed to streamline the process of building, deploying, and managing machine learning (ML) pipelines on Google Cloud’s Vertex AI platform. Its primary goal is to improve efficiency for data science teams by reducing the time it takes to deploy ML models while observing MLOps best practices.

 

Overview 

Conduit was designed to offer the following key benefits: 

  • Expedite the migration of on-premises ML pipelines. 

  • Reduce the time and effort required to operationalize ML models. 

  • Simplify the creation of new ML pipelines for most data science use cases. 

  • Increase efficiency by facilitating a high reuse rate of basic ML pipeline components (e.g., data ingestion, model training, endpoint ingestion, etc.). 

  • Foster and support machine learning governance by promoting MLOps best practices. 

 

Why use Conduit? 

In the absence of Conduit, data scientists at CVS Health would need to work with KubeFlow Pipelines (KFP) or TensorFlow Extended (TFX) APIs to create machine learning pipelines in Vertex AI. Both of these processes are time consuming and have steep learning curves; in addition, these methods make it difficult to reuse custom pipeline components across CVS Health. 

Conduit was introduced to not only address these key efficiency issues, but to promote a healthier user focus on what really matters: exploring data and building great models. With Conduit, data scientists can create fully functional Vertex AI pipelines by answering a straightforward TUI-based step-by-step questionnaire. 

The result is rapidly produced pipelines composed of reusable and standardized CVS Health components that can be run or scheduled on Vertex AI. 

 

Audience 

Our expected user base for Conduit are Data Scientists and ML Engineers who are building supervised Regression and Classification models using Python.  

While a wide variety of potential use cases and classifications exist for this intended audience, the A&BC team anticipates that the following scenarios are the most representative: 

  • Users who want to operationalize new ML models without the need to use KubeFlow and TensorFlow APIs. 

  • Users who want to easily move existing use cases from on-premises clusters to Google Cloud Platform. 

  • Users who want to leverage the stability and accuracy of existing pipeline components as they build their own. 

  • Users who want the reliability of a guided experience while ensuring that their ML models are governable, observable, and explainable. 

Before you start

Before installing Conduit, make sure your project adheres to the following criteria to determine if Conduit is a useful tool for your needs:

  • The required underlying data is in Big Query.

  • Python was used for model development.

  • The ML Framework is either sklearn, xgboost, tensorflow, or pytorch.

  • The model is set up as a [supervised] classification or regression problem.

  • CPUs were used for training (as opposed to GPUs).

If your project does not meet the above basic criteria, use a different tool for deployment.

If your project does meet the basic criteria above, the following additional prerequisites must be in place before proceeding to install Conduit:
